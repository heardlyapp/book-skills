# How Algorithms Discriminate — Core Framework

The central argument: search engines are not neutral information retrieval systems. They are commercial platforms that produce and reproduce social inequality.

## The Three Layers of Algorithmic Oppression

**Layer 1: Data.** Algorithms learn from data. If the data contains racist patterns, the algorithm will reproduce them.

**Case: The Internet Is Biased.** Google's algorithm is trained on web pages. The web is created by people who live in a racist society. The algorithm learns the biases of its trainers and amplifies them at scale.

**Case: The Feedback Loop.** When users click on results, the algorithm learns that those results are relevant. If racist results get clicks, the algorithm shows more racist results. The cycle reinforces itself.

**Layer 2: Commercial Priorities.** Google's primary goal is revenue. Search results are shaped by advertising and commercial partnerships.

**Case: The AdWords System.** Google's AdWords program allows advertisers to bid for placement. The highest bidder gets the top result — regardless of relevance or accuracy.

**Case: The Quality Score.** Google rates the "quality" of ads based on expected click-through rates. This system can penalize content that is less likely to generate clicks, including content about marginalized communities.

**Layer 3: Classification Systems.** How search engines categorize information determines what is findable. Classification is an exercise of power.

**Case: The Classification of "Black Girls."** Google's algorithm categorized "black girls" in proximity to pornography because of how the web linked these terms. The classification was not neutral — it was the product of a system that devalues Black women and girls.

**Case: The Classification of "Professional Women."** Searches for "professional women" return mostly white women. The classification system has learned to associate "professional" with "white" and not with "Black."
