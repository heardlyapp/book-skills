## Practical Techniques

### 1. The Gulf Analysis

When evaluating a product, identify the Gulf of Execution: "Can the user figure out what to do?" and the Gulf of Evaluation: "Can the user figure out what happened?" If either gulf is wide, the design needs work.

### 2. The Affordance Audit

Look at every design element and ask: "What does this object afford?" If the answer conflicts with the intended action, the design is wrong. A door should afford its correct action.

> **Case: The Glass Door** (Chapter 1): A glass door has no handle, no plate, no sign. It affords nothing. Users walk into it. The affordance (push or pull?) is invisible. A vertical metal strip on one side would make the affordance clear — and prevent injury.
> **Key takeaway**: Invisible affordances are the cause of many design failures. Make the action visible.

### 3. The Error Recovery Test

Try using a product and intentionally make a mistake. Can you recover? How many steps does it take? The best designs make error recovery easy and obvious.

### 4. The Conceptual Model Check

Ask: "What does the user think is happening?" If their mental model does not match the system's design, confusion results. A clear conceptual model should be discoverable through interaction.

> **Case: The Digital Watch** (Chapter 3): Norman describes a digital watch with too many buttons doing too many things with no clear conceptual model. Users could not figure out how to set the time. The design violated every principle: no mapping, no signifiers, no conceptual model.
> **Key takeaway**: A product that requires a manual to use is a design failure. If the conceptual model is not obvious from the design, the design must change.
